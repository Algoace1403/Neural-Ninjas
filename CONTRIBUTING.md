# Contributing

Thanks for your interest in improving this project! Contributions are welcome.

## Getting started

1. Fork the repo and clone your fork.
2. Create a feature branch: `git checkout -b feat/my-thing`.
3. Make your changes, following the style used in the existing code.
4. Commit with a clear message (`feat:`, `fix:`, `docs:`, `chore:`, etc.).
5. Push and open a pull request.

## Code style

- Keep functions small and named for what they do.
- Add comments for non-obvious logic.
- Match the formatting of nearby code.

## Running locally

See the README for setup and usage instructions.

## Questions

Open an issue if you are unsure about anything — happy to help.
